# GamePrince 全生命周期开发任务大盘 (Roadmap & Backlog)

> **核心指挥部备忘录**：
> 本清单由“项目核心总管理”视角统一维护。
> **神圣铁律**：无论功能如何演进，GamePrince 对目标 Godot 项目始终保持 **绝对只读 (Read-Only)** 与 **零侵入 (Zero-Intrusion)**。所有分析、建议、警告均在外部沙盒中呈现，确保证项目工程绝对安全。

---

## 纪元 I：全能敏捷底座构建 [📍 当前聚焦阶段]
*目标：补齐项目管理的基础闭环，从“看板备忘”跨越至“专业敏捷管理”。*

### 1.1 里程碑与版本管线 (Milestone & Release)
- [x] **里程碑数据模型**：设计 `Milestone` 数据结构，包含版本号、起止时间、目标描述。
- [x] **关联与聚合**：Task 支持绑定至特定 Milestone。
- [x] **版本追踪视图**：新增 Milestone Dashboard，可视化当前版本的进度条（Completed / Total Tasks）。
- [ ] **燃尽图 (Burndown Chart)**：基于 Milestone 周期推演并绘制简单的进度燃尽趋势。

### 1.2 沉浸式富文本体验 (Markdown Engine)
- [x] **Markdown 解析引擎**：引入或实现轻量级 Markdown 渲染器至 Task 描述区域。
- [x] **语法高亮与渲染**：支持加粗、列表、代码块高亮展示。
- [x] **交互式代办项**：Task 内部描述中的 `[ ]` 任务框支持直接点击勾选并保存。

### 1.3 无感时间流审计 (Time Tracking) [x]
- [x] **工时预估模型**：Task 实体新增 `EstimatedHours` 与 `LoggedHours` 字段。
- [x] **内置随身计时器**：在 Task 面板添加 Start/Stop 计时按钮，精确追踪实际开发时间。
- [ ] **个人工时周报**：按周统计并可视化打点记录的时间块占比。

### 1.4 多模视图联动 (Views Expansion)
- [ ] **列表视图 (List/Grid)**：提供紧凑的表格形态，便于批量编辑和速览。
- [ ] **日历阵列 (Calendar)**：结合任务的 `DueDate` 和时间打点，映射至可视化日历中。

## 纪元 II：Godot 引擎核心透视仪 [🔜 下一阶段]
*目标：赋予软件独家的 Godot 资产解析能力，打造非侵入式“上帝之眼”。*

### 2.1 引擎元数据识别网
- [ ] **`project.godot` 核心探针**：读取项目名称、Godot 大小版本号、主场景入口等关键参数。
- [ ] **多模态发版配置侦测**：静默解析 `export_presets.cfg`，列出已配置的打包平台与打包参数。

### 2.2 资源星系图谱 (Asset Topological Graph)
- [ ] **静态解析树**：在不启动 Godot 的前提下，纯文本解析 `.tscn` 与 `.tres` 引用的 `ext_resource` 依赖链。
- [ ] **依赖引用可视化**：绘制核心预制体 (Prefab/Scene) 的依赖关系列表及嵌套深度。

### 2.3 上帝之眼：死区回收与瘦身建议
- [ ] **全量资产快照**：遍历工程下所有 `.png`, `.wav`, `.ogg`, `.tscn`, `.gd` 等文件集。
- [ ] **孤岛巡游者 (Orphan Detector)**：比对依赖链与文件集，找出未被任何场景或代码显式引用的“幽灵资源”。
- [ ] **瘦身长清单导出**：生成冗余资源清单（强调仅提供参考，需开发者在引擎内手动确认清除）。

### 2.4 代码巨兽雷达 (Code Smell Radar)
- [ ] **行数阀值扫描**：全工程 `.gd` 脚本粗略行数统计，高亮显示超大（如 >800 行）的神级类 (God Class)。
- [ ] **复杂度初步嗅探**：通过正则匹配 `func` 关键字密度，标记潜在的过度臃肿对象。

## 纪元 III：自动化代码纪律委员 [🔭 中期规划]
*目标：构筑自动化防备线，拦截因低级错误造成的管线阻塞。*

### 3.1 跨端导出排雷器
- [ ] **大小写雷区提示**：扫描因为资源命名大小写不一致可能导致的 Windows 可用但 Linux/导出崩溃的隐患。
- [ ] **非常规字符探测**：扫描文件名中包含空格或中文字符的节点，发出跨平台不兼容预警。

### 3.2 Git 拦截者护城河
- [ ] **规范挂载 (Git Hooks)**：提供一键为根目录安装 `pre-commit` 钩子的快捷功能。
- [ ] **Commit 格式强制检验**：确保团队/个人的提交信息符合 Conventional Commits（如 `feat:`, `fix:` 等）。

### 3.3 架构环形地图 (Architecture X-Ray)
- [ ] **继承链脉络图**：解析 `.gd` 中的 `extends` 关键字，绘制项目独有的继承森林。

## 纪元 IV：中台知识沉淀与数据归境 [🌌 长线愿景]
*目标：在项目收尾期与维护期，沉淀技术资产，形成开发留存。*

### 4.1 极速内部百科 (Private Wiki)
- [ ] **脱离代码的架构白皮书**：提供对当前项目系统设计的 Markdown 知识库托管（存在 `.gameprince/docs` 中）。
- [ ] **API 备忘速查卡**：记录核心 Global 单例的契约接口备忘。

### 4.2 杀青总结大盘 (Wrap-up Report)
- [ ] **全息成就长图生成**：一键生成本项目的最终总代码量、最活跃开发时段、解决总任务数、Commit 里程碑等图文并茂的“成就战报”。
- [ ] **数据封存归档**：将项目看板与时间轴序列化出只读 HTML 以供永久纪念。
